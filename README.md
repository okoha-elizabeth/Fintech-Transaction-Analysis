# Fintech-Transaction-Analysis

## Project Overview

This project analyzes digital transaction data to evaluate system performance, identify key drivers of transaction failures, and understand customer channel preferences. The goal is to improve transaction reliability and enhance user experience.

## Objective

To identify the causes of transaction failures, analyze channel usage patterns, and evaluate overall transaction success rates to support operational improvements in a fintech environment.

## Tools Used

1. Python (Pandas, Matplotlib, Seaborn)
2. Excel
3. Power BI

## Data Source

This is a synthetic dataset generated for practice purpose only to simulate real-world fintech transaction scenarios. 

## Business Questions

1. What causes transaction failure?
2. Which channel is mostly used?
3. What is the transaction success rate?

## Key Insights

1. Transaction failures are concentrated around specific factors:
2. ATM transactions account for the highest failure rate (50%)
3. Mobile transactions contribute significantly to failures (40%)
4. Abuja records all observed failures, indicating a location-specific issue
5. Android devices account for all failures, suggesting possible compatibility or performance issues
6. Withdrawal transactions have higher failure rates compared to payments
7. Mobile is the dominant transaction channel, accounting for 66.6% of total transactions, highlighting strong user preference for digital platforms
8. The overall transaction success rate is 83.3%, indicating relatively high performance but still leaving room for improvement in reliability

## Business Recommendations

1. Improve ATM infrastructure and network reliability to reduce failure rates
2. Optimize mobile application performance, particularly for Android devices
3. Investigate and resolve location-specific issues in Abuja
4. Enhance system reliability for withdrawal transactions
5. Prioritize investment in mobile platforms due to high user adoption
6. Implement real-time monitoring systems to detect and resolve transaction failures quickly
7. Set performance targets to improve success rate beyond current levels

## Deliverables

1. Data Cleaning and Preparation 
2. Exploratory Data Analysis (EDA)
3.Business Insights and Recommendations
4.Power BI Dashboard

## Conclusion

The analysis reveals that transaction failures are driven by specific channels, devices, and locations rather than random occurrences. Addressing these targeted issues and strengthening system performance will significantly improve transaction reliability and customer trust.
